# 用户域名允许列表与自动填充匹配规则设置 - 产品需求文档

## Overview
- **Summary**: 将"域名允许列表"和"自动填充匹配规则"功能从管理员设置迁移到用户设置，使每个用户可以独立配置自己的自动填充域名规则。
- **Purpose**: 允许用户自主控制哪些域名可以自动填充密码、哪些域名需要阻止自动填充，以及匹配模式的选择，提升个性化体验和安全性。
- **Target Users**: 所有普通用户（非管理员）

## Goals
- 将域名允许列表、域名阻止列表、自动填充匹配规则从管理员设置移至用户设置
- 每个用户可以独立配置自己的域名规则和匹配模式
- 保持原有功能的完整性和交互体验一致
- 管理员设置页面不再包含这些用户级功能

## Non-Goals (Out of Scope)
- 不改变域名规则的存储方式（仍使用 localStorage）
- 不添加新的域名规则类型
- 不改变自动填充的核心逻辑

## Background & Context
目前"域名允许列表"、"域名阻止列表"和"自动填充匹配规则"功能位于管理员设置页面 `/admin/settings`，属于全局配置。但这些实际上是用户级别的设置，应该让每个用户独立配置。

## Functional Requirements
- **FR-1**: 用户设置页面新增"域名允许列表"管理功能，支持添加、删除域名
- **FR-2**: 用户设置页面新增"域名阻止列表"管理功能，支持添加、删除域名
- **FR-3**: 用户设置页面新增"自动填充匹配规则"设置，支持精确匹配和模糊匹配切换
- **FR-4**: 管理员设置页面移除域名允许列表、域名阻止列表和自动填充匹配规则功能
- **FR-5**: 用户设置中的域名规则使用用户级 store 状态管理，与管理员设置隔离

## Non-Functional Requirements
- **NFR-1**: UI 设计与现有用户设置页面风格保持一致
- **NFR-2**: 操作反馈清晰（添加/删除成功提示）
- **NFR-3**: 设置变更即时生效

## Constraints
- **Technical**: 基于现有 React + Zustand + Tailwind CSS 技术栈
- **Dependencies**: 需要修改 types/index.ts、store/index.ts、Settings.tsx、AdminSettings.tsx

## Assumptions
- 用户设置页面已有完整的侧边栏导航结构
- 用户级 store 已有 `SettingsState` 切片基础
- 域名规则类型 `DomainConfig` 已在 types/index.ts 中定义

## Acceptance Criteria

### AC-1: 用户设置页面显示域名允许列表
- **Given**: 用户登录并访问设置页面
- **When**: 用户切换到"安全设置"或新的"自动填充设置"板块
- **Then**: 页面显示域名允许列表，包含添加域名输入框和现有域名列表，每个域名有删除按钮
- **Verification**: `human-judgment`

### AC-2: 用户可以添加域名到允许列表
- **Given**: 用户在域名允许列表输入框中输入域名（如 `*.google.com`）
- **When**: 用户点击"添加"按钮或按回车键
- **Then**: 域名被添加到允许列表，显示成功提示，输入框清空
- **Verification**: `human-judgment`

### AC-3: 用户可以从允许列表删除域名
- **Given**: 允许列表中存在域名
- **When**: 用户点击域名旁边的删除按钮
- **Then**: 域名从列表中移除，显示成功提示
- **Verification**: `human-judgment`

### AC-4: 用户设置页面显示域名阻止列表
- **Given**: 用户登录并访问设置页面
- **When**: 用户切换到"安全设置"或新的"自动填充设置"板块
- **Then**: 页面显示域名阻止列表，包含添加域名输入框和现有域名列表，每个域名有删除按钮
- **Verification**: `human-judgment`

### AC-5: 用户可以添加/删除阻止列表域名
- **Given**: 用户在阻止列表输入框中输入域名或点击删除按钮
- **When**: 用户执行添加或删除操作
- **Then**: 操作成功，列表更新，显示相应提示
- **Verification**: `human-judgment`

### AC-6: 用户可以切换自动填充匹配模式
- **Given**: 用户在设置页面看到匹配模式选项
- **When**: 用户点击"精确匹配"或"模糊匹配"按钮
- **Then**: 匹配模式切换，显示成功提示，按钮状态更新（高亮当前选中项）
- **Verification**: `human-judgment`

### AC-7: 管理员设置页面不再包含域名规则功能
- **Given**: 管理员登录并访问管理员设置页面
- **When**: 管理员浏览页面内容
- **Then**: 页面中不显示域名允许列表、域名阻止列表和自动填充匹配规则相关内容
- **Verification**: `human-judgment`
